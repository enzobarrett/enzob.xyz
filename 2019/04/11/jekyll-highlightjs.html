Using highlight.js with Jekyll

Apr 11, 2019 

{
  "format"        : "syntax highlighting example",
  "language"      : "json",
  highlight.js: 
  {
    "languages"   : 185,
    "styles"      : 89,
    "dark-styles" : 51
  },
  rogue:
  {
    "languages"   : 100,
    "styles"      : 20,
    "dark-styles" : 5
  }
}

Itâ€™s 2019 and everybody seems to be using a dark IDE; why not have dark code snippets on your site? There are many syntax highlighting options for Jekyll, the main one being Rogue. Highlighting with Rogue works well and is easy, but there are benefits in using highlight.js instead. Advantages that highlight.js has over Rouge are as follows:

  • More languages supported
  • Far more styles
  • The ability to pick and choose what languages you include
  • Automatic language detection
  • Better documentation
  • Much better dark themes

I personally did not like any of the pre-made dark themes for rogue. There is the ability to make your own but why do that when there are so many great themes already out there?

How to setup highlight.js

Source the files

You are going to need to source the highlight.js files in every post. First, you want to create a file to override the default header. To override a theme file a directory and file need to be created with the same names and the file will override the theme default.

# cd to your Jekyll project directory
cd $YOURDIRECTORY

# first make a new directory in your Jekyll folder to override the proper file
mkdir _includes

# find the directory the default files are in
bundle show minima

# cp the header file
cp $YOURLOCATION/_includes/head.html _includes/
